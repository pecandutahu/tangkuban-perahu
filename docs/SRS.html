Software Requirements Specification (SRS)

Nama Perangkat Lunak: Sobat Gaji (Payroll Trucking System)

Versi Dokumen: 1.0

Platform: Aplikasi Berbasis Web

1. Pendahuluan

1.1 Tujuan

Dokumen SRS ini memberikan deskripsi komprehensif mengenai spesifikasi teknis dan fungsionalitas perangkat lunak Sobat Gaji. Dokumen ini digunakan sebagai referensi teknis yang dapat dibaca dan diperbarui di sepanjang siklus pengembangan aplikasi.

1.2 Lingkup Perangkat Lunak

Perangkat lunak ini dibangun menggunakan arsitektur modern dengan backend Laravel 11 (PHP 8.2+) dan frontend SPA (Single Page Application) menggunakan Vue 3 (Composition API) yang dihubungkan melalui Inertia.js. Sistem ini menangani perhitungan payroll khusus trucking (Gaji Pokok, Uang Jalan, Klaim) ditambah kalkulasi BPJS Kesehatan & Ketenagakerjaan otomatis secara proporsional serta PPh 21 TER 2024.

2. Kebutuhan Fungsional (Functional Requirements)

2.1 Modul Autentikasi & Keamanan

FR-01Sistem mengimplementasikan proteksi login Brute-Force menggunakan validasi Math Captcha di sisi server.
FR-02Sistem mengadopsi struktur Role-Based Access Control (RBAC) memanfaatkan package spatie/laravel-permission untuk mengamankan route dan fitur antarmuka berdasarkan role pengguna (Super Admin, Admin, Finance, Operator).

2.2 Modul Master Data Utama

FR-03Manajemen Cabang: Mendukung CRUD data cabang dan nilai umr_amount (berfungsi sebagai threshold kalkulasi asuransi sosial BPJS).
FR-04Manajemen Karyawan: Mengelola data lengkap termasuk relasi ke Cabang, Jabatan, Status PTKP, dan menyimpan besaran dasar Gaji Pokok (GP).
FR-05Manajemen Komponen & Template: Admin dapat mendefinisikan kamus Komponen (Earning/Deduction) dan menggabungkannya ke dalam Template Gaji berdasarkan jabatan/peran.

2.3 Modul BPJS & Perhitungan Otomatis

FR-06Sistem menyediakan BpjsCalculatorService yang otomatis menghasilkan 6 komponen standar (JHT, JP, Kes untuk Karyawan dan Perusahaan) ketika form karyawan baru/update dikirim dari antarmuka (Hybrid Approach).
FR-07Dasar pengali BPJS secara dinamis membandingkan Gaji Pokok karyawan dan UMR cabang, dan menggunakan nilai yang paling besar.
FR-08Sistem menyediakan perintah Command Line php artisan payroll:sync-bpjs untuk meregenerasi (sync) data potongan BPJS seluruh karyawan apabila terjadi perubahan regulasi persentase tarif secara global.

2.4 Modul PPh21 & Eksekusi Payroll (Generation)

FR-09Kalkulator PPh21 terintegrasi pada fase Generate Payroll, menghitung pajak berdasar Tarif Efektif Rata-rata (TER) dan nilai Gross komponen yang ditandai kena pajak.
FR-10Proses Generate Payroll menyalin susunan Template Gaji dan Komponen Override karyawan untuk dicetak secara permanen ke dalam tabel riwayat payroll_slips.
FR-11Fungsionalitas Locking: Periode dan slip gaji yang telah diselesaikan dapat dikunci (Locked/Published), sehingga imun dari modifikasi eksternal.

3. Kebutuhan Non-Fungsional (Non-Functional Requirements)

  • Infrastruktur Frontend: Menggunakan Tailwind CSS untuk penataan visual gaya modern, efisien, dan responsif. Interaktivitas difasilitasi oleh komponen-komponen Vue 3.
  • Continuous Integration / Deployment: Aplikasi memiliki kapabilitas automasi testing (dengan database memori SQLite via phpunit.xml) dan auto-deployment ke environment VPS berbasis Linux (Ubuntu) yang diorkestrasi melalui GitHub Actions Workflow.
  • Integritas Data: Menggunakan validasi Form Request dari sisi backend Laravel untuk menjamin bahwa data masukan angka (gaji pokok, uang jalan) diformat dengan benar sebelum masuk database PostgreSQL/MySQL.

4. Pemetaan Struktur Basis Data (Database Structure)

Skema basis data relasional dikelola secara komprehensif menggunakan sistem migrasi Laravel:

  • employees: Data demografi, finansial dasar, dan FK untuk PTKP, branch, dan role.
  • branch: Entri multi-lokasi termasuk umr_amount.
  • settings: Kumpulan konfigurasi dinamis (persentase JKK, versi PPh21 aktif, dsb).
  • payroll_components & payroll_templates: Arsitektur penyusun komponen slip gaji.
  • employee_components: Komponen individual dan override, termasuk penyimpan hasil potong BPJS hybrid.
  • payroll_periods & payroll_slips: Perekaman snapshot payroll setiap siklus (bulanan).

5. Antarmuka Sistem (System Interfaces)

  • Web Frontend (User Interface): Dibangun menggunakan komponen Inertia.js untuk menghadirkan pengalaman navigasi SPA yang mulus tanpa refresh halaman saat mengelola payroll.
  • Artisan Console Interface: Menyediakan custom command untuk tugas maintenance administratif seperti app:clean-demo dan sinkronisasi data master.
